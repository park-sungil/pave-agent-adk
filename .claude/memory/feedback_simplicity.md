type: feedback

langgraph에서 하나의 파이프라인으로 여러 일을 처리하려다 보니 의도 분류(5개), 분석 힌트(7개), 엔티티(13개) 등이 과도하게 늘어났다. ADK에서는 오케스트레이터가 자유롭게 도구를 조합할 수 있으므로, 이런 분류 체계를 코드에 하드코딩할 필요가 없다.

**Why:** 유지보수가 어려워지고, 사용자가 따라잡기 힘듦.
**How to apply:** 기능 추가 시 최소 단위로. "이게 정말 필요한가?" 먼저 확인. instruction은 간결하게, 복잡한 분류 체계 대신 예시 기반으로.
